三、GPIOD 端口问题 — 确认
======================================================================

  多个独立测试确认 GPIOD 不可用于 GPIO 输入:
    - PD6: 硬件 USART1 RX 不通 | SW UART 外接 CH340 → edge=0
    - PD4: 外接 CH340 → 系统崩溃/调试串口卡死
    - 同板 MCU 自回环可用 (PA2 推挽强驱→PD6), 但外接信号失败
    - 输出正常: PD4=步进 DIR 工作中
    - 历史佐证: laser.h "TOF200F 不可使用 PD6 作为 RX"

  结论: GPIOD 全端口避开输入用途, 仅做输出或释放

四、SW UART 软件串口 — 最终方案
======================================================================

  新建文件: App/inc/sw_uart.h, App/src/sw_uart.c
  方案: TIMER1 @ 345.6kHz (3x 过采样), 两路全双工, 115200 8N1
  ISR: gd32h7xx_it.c TIMER1_IRQHandler → SwUart_ISR()

  【最终引脚】
    CH1: PE2(J5-56)=TX + PE5(J5-58)=RX  (GPIOE, 同端口相邻)
    CH2: PF7(J4-17)=TX + PF6(J4-19)=RX  (GPIOF, 同端口相邻)

  【软件修复 (入版)】
    P1 - RX 时序: start bit 验证从 1tick→2tick, D0 采样从 3→4tick
    P2 - RX 上拉: GPIO_PUPD_NONE → GPIO_PUPD_PULLUP
    P3 - TX 诊断: sw1/2_tx_timeout 超时计数器
    P4 - printf: %lu→%u (MicroLIB 兼容)

  【PIT 时钟注意】
    pit_init 按 SystemCoreClock(600MHz) 折算参数
    TIMER1 在 APB2(300MHz), 需传 2x 期望周期: 1736→实际868→345.6kHz

  【验证】PC 回显测试通过, 两路全双工稳定

  【已弃用】
    PA2(J4-21) + PD6(J4-23) — 原 CH1, PD6 不可用, 已释放

七、GPS/北斗定位模块 — ATGM336H-5N 新增
======================================================================

  新建文件: App/inc/gps.h, App/src/gps.c
  参考: ATGM336H-5N 模块资料 (D:\BaiDuNetdiskDownload\ATGM336H-5N模块资料)

  【模块参数】
    型号: ATGM336H-5N-1X
    卫星: GPS + BDS + GLONASS + 伽利略
    默认波特率: 9600, 8N1
    协议: NMEA-0183
    供电: 3.3~5V
    定位精度: 2.5m (CEP50)
    更新率: 1Hz

  【硬件接线】
    GPS模块          CIMC 核心板 (SW UART CH1)
    VCC (5V)    →    5V
    GND         →    GND
    TX          →    PE5 (J5-58, SW UART1 RX)
    RX          →    PE2 (J5-56, SW UART1 TX)
    PPS         →    NC (可选 GPIO)

  【SW UART 波特率变更】
    原 115200 (TIMER1 @ 345.6kHz, period=1736)
    现 9600   (TIMER1 @ 28.8kHz,  period=20834)
    原因: ATGM336H-5N 默认 9600，GPS 模块数据量大且连续输出
    影响: 两路 SW UART 均为 9600bps

  【NMEA 解析】
    重点解析 $GNRMC (兼容 $GPRMC/$BDRMC)
    辅助解析 $GNGGA (卫星数 + 定位质量)
    提取字段:
      - UTC时间 (hhmmss.ss) → 拆分为时分秒
      - 定位状态 (A=有效/V=无效)
      - 纬度 ddmm.mmmm → 十进制度 (度 + 分/60)
      - 经度 dddmm.mmmm → 十进制度
      - 对地航速 (节) → m/s
      - 对地航向 (度)
      - UTC日期 (ddmmyy)
    坐标转换: ddmm.mmmm → 十进制度 = 度整数部 + 分/60

  【API】
    GPS_Init()         — 初始化, 清空数据结构 + 排空 SW UART
    GPS_Process()      — 运行时处理, 在主循环每次迭代调用
    GPS_IsFixValid()   — 检查定位是否有效
    GPS_PrintStatus()  — 打印单行状态 (定位/时间/经纬度/速度/航向/卫星数)

  【测试模式: GPS_MODULE_TEST】
    功能: SW UART CH1 接收 GPS 模块 NMEA 数据, 解析后每秒打印状态
    验证: 模块板载 LED 闪烁 = 已定位; 串口打印 FIX OK + 坐标

  【正常模式集成】
    System_Run() 初始化阶段: SwUart_Init() → GPS_Init()
    主循环 (50Hz): GPS_Process() 每次迭代调用 (非阻塞, 逐字节喂入)
    1Hz上报: GPS_PrintStatus() 输出定位/速度/航向等数据

  【MicroLIB 兼容】
    - 避免使用 atof/atoi (ARM Compiler 5 MicroLIB 可能不支持)
    - 自实现 nmea_atof/nmea_atoi 简化解析函数
    - printf 浮点格式: (double)强制转换 float 参数

八、机械臂示教模式 — ARM_TEACH_TEST
======================================================================

  【目的】纯机械臂调试环境，不初始化电机/MPU6050/TOF/GPS/步进/SW UART
  【启用】main.c 顶部 #define ARM_TEACH_TEST
  【初始化】PS2 → ServoArm(PCA9685+I2C) → TIMER3 50Hz

  【PS2 按键布局】
    ←/→      底座旋转 ch0      (270°舵机, 0~270°)
    ↑/↓      大臂俯仰 ch1      (180°舵机, 0~180°)
    □/○      小臂俯仰 ch2      (180°舵机, 0~180°)
    △/×      腕部俯仰 ch3      (180°舵机, 0~180°)
    L1/L2    腕部旋转 ch4      (180°舵机, 0~180°)
    R1/R2    夹爪开合 ch5      (安全限位 51°~141°)
    START    捕获当前姿态 → 串口输出动作表格式代码
    SELECT   归位 PARK (135,90,90,30,90,141)

  【速度控制】三个参数独立可调
    ServoArm_SetSpeed(divider)          遥控手动: 50/divider °/s, 默认10→5°/s
    ServoArm_SetSmoothSpeed(deg)        归位步长: 1~10 °/帧, 默认1
    ServoArm_SetSmoothDivider(div)      归位分频: 1=50Hz~20=2.5Hz, 默认2
    当前归位实际速度: 1°/帧 ÷ 2 = 25°/s

  【示教流程】
    1. 用 PS2 遥控器手动摆好机械臂姿态
    2. 按 START → 串口输出动作表格式代码 { 135, 90, 90, 30, 90, 141 }
    3. 重复收集所有动作
    4. 将代码填入 action_angle[][] 动作表

九、调试串口波特率调整 — 115200 → 9600 (2026-06-19)
======================================================================

  【修改原因】统一系统波特率，与 SW UART (GPS 9600) 保持一致，降低无线干扰敏感度

  【修改文件】
    User/main.c:60           uart_init(UART_DBG, 115200) → 9600 (实际生效)
    User/main.c:469          printf 提示信息 @ 115200 → @ 9600
    App/src/ps2.c:279        函数注释中的波特率引用
    App/inc/ps2.h:147        函数注释中的波特率引用

  【影响说明】
    - printf 输出速度降低: 1 字节 ≈ 1ms @ 9600 (原 0.087ms @ 115200)
    - 建议保持 1Hz 上报频率，避免阻塞 50Hz 主循环
    - 串口助手需同步改为 9600 8N1
