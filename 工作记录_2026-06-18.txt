一、步进电机传送带驱动 — 新增模块
======================================================================

  【参考】STEP 工程 (D:\学习\电赛\2025\STEP)，使用相同的步进电机驱动板

  【STEP 工程驱动分析】
    - 接口: 3 线 (EN/DIR/STEP)，GPIO 比特位控制
    - 脉冲: 定时器中断 → 置 DIR → STEP↑ → STEP↓
    - 速度: 分频器 (speed_div) 控制步进频率
    - 参数: 1.8° 全步, 1/16 微步 → 3200 步/圈
    - TIM3 @ 500μs 驱动偏航轴, TIM4 @ 1ms 驱动俯仰轴

  【本工程实现】
    - 新建 App/inc/step_motor.h — 驱动头文件
    - 新建 App/src/step_motor.c — 驱动实现
    - 修改 User/gd32h7xx_it.c — 添加 TIMER6_IRQHandler
    - 修改 User/main.c — System_Run 中集成

  【引脚分配】(均为 J4 空闲引脚)
    PA2 (J4-21) → STEP  步进脉冲
    PD4 (J4-22) → DIR   方向 (H=正转, L=反转)
    PD6 (J4-23) → EN    使能 (H=使能)

  【定时器】TIMER6 — 500μs (2kHz) 中断

  【API】
    Stepper_Init()              — GPIO + TIMER6 初始化
    Stepper_SetSpeed(dir, div)  — dir: 1/-1/0, div: 1(快)~200(慢)
    Stepper_GetCurStep()        — 累计步数 (调试)

  【EN 引脚逻辑】
    - 初始化: EN=H (使能)
    - 运行中: 始终保持 EN=H (与 STEP 工程一致)
    - 停止时: 仅停止脉冲, EN 不变

六、硬件串口测试（USART1 / UART6）— 失败
======================================================================

  【测试方法】
    - 在 uart_driver 中新增 UART_USART1(PA2/PD6,AF7) 和 UART_UART6(PF7/PF6,AF6)
    - 添加 USART1_IRQHandler / UART6_IRQHandler
    - 编写 UART_ECHO_TEST 回环测试

  【测试结果】
    USART1 (PA2/PD6):
      - TX ✓: PC 能收到 "USART1 OK" / "U1 heartbeat"
      - RX ✗: PC 发送数据，MCU STAT 寄存器 RBNE 始终为 0
      - PA2→PD6 杜邦线物理回环: rx 计数仍为 0
      → 结论: PD6 管脚无法作为 USART RX

    UART6 (PF7/PF6):
      - TX ✗: 上电只发出两个 0x00，之后无输出
      - RX ✗: PC 发送数据，MCU 无响应
      → 结论: PF6/PF7 管脚 UART 功能在这块板上有底层问题

  【寄存器诊断】
    USART CTL0=0x2D (UEN=1,REN=1,TEN=1) ✓
    GPIO CTL: PD6=AF, PF6=AF, PF7=AF ✓
    BAUD: 1302 (150MHz/1302≈115200) ✓
    CLK: APB1=150MHz ✓
    配置层面一切正确，是管脚/板级层面的问题

  【历史佐证】
    laser.h 注释: "TOF200F 模块不可使用 PD6 作为 RX 引脚（已验证通信失败）"
    与本次测试结论一致

七、软件 UART 实现 (sw_uart) — TX 通, RX 不通
======================================================================

  【方案】
    - TIMER1 @ 3x 波特率 (345.6kHz) 驱动两路全双工状态机
    - TX: 精确位定时，ISR 中移位输出
    - RX: 下降沿检测 + 3 倍过采样
    - 引脚: CH1=PA2(TX)+PD6(RX), CH2=PF7(TX)+PF6(RX)
    - 纯 GPIO 读写，不依赖硬件 USART AF

  【测试结果】
    - TX ✓: PC 能收到 "SW UART1 OK" / "SW UART2 OK"
    - RX ✗: 发送字符无回显，E(下降沿计数)=0
    - ISR 正常运行 (sw_isr_count 快速增长)

  【关键发现】
    - PD6/PF6 作为 GPIO 输入，ISR 中采样 GPIOD_ISTAT/GPIOF_ISTAT
    - 下降沿检测始终为 0 → CH340 TX 的电平变化未被 MCU 识别
    - 与硬件串口测试结论一致: GPIOD/GPIOF 的输入功能在此板上有问题

  【PIT 时钟陷阱】
    pit_init 参数按 SystemCoreClock(600MHz) 折算，TIMER1 在 APB2(300MHz)
    需传 2x 期望周期: period=1736 → 实际 868 → 345.6kHz

十、SW UART 软件修复 + 硬件调试记录 (2026-06-18 第二~五轮)
======================================================================

  【软件修复 (已完成并验证)】
    P1 - RX 3x 过采样时序修正:
      问题: 下降沿检测后只等 1 tick 就验证 start bit, D0采样在3tick
            而 D0 中心在 4.5tick 处, 偏了半个 bit 周期
      修复: state 1 中改等 2 ticks 再验证, D0采样在 4 tick 处
            误差从 ±1.5 tick 缩小到 ±0.5 tick (±1/6 bit)
      文件: App/src/sw_uart.c case 1: tick==1 → tick==2

    P2 - RX 引脚内部上拉:
      GPIO_PUPD_NONE → GPIO_PUPD_PULLUP (两路 RX 均加)
      防止浮空引脚读不到 HIGH 导致边缘检测永不触发

    P3 - TX 超时诊断:
      新增 sw1_tx_timeout / sw2_tx_timeout
      SendByte 忙等超时时递增, 1Hz 统计包含 TO= 字段

    P4 - printf 格式统一 %u (避坑 MicroLIB %lu)

======================================================================

  【CH1 — GPIOE (PE2=TX, PE5=RX)】
    PE2 (J5-56): 推挽输出, 空闲 HIGH
    PE5 (J5-58): 上拉输入, GPIO_PUPD_PULLUP
    同端口 RCU_GPIOE, 相邻 5mm, 无模块冲突
    ✅ 回环测试通过, ✅ 外接 CH340 PC 回显测试通过

  【CH2 — GPIOF (PF7=TX, PF6=RX)】
    PF7 (J4-17): 推挽输出
    PF6 (J4-19): 上拉输入
    同端口 RCU_GPIOF, 相邻 5mm
    ✅ 回环测试通过, ✅ 外接 CH340 PC 回显测试通过

  【已弃用引脚】
    PA2/PD6 — 原 CH1, PD6(GPIOD) 外部信号不通, 已释放
    PE4/PE6 — 曾候选, 与摄像头 OV2640 DCI 冲突, 不用

  【GPIOD 端口判决 (2026-06-18 确认)】
    - PD6: 硬件 USART1 RX 不通 | SW UART 外接 CH340 → E=0
    - PD4: 外接 CH340 → 系统崩溃/调试串口卡死
    - 输出正常 (步进电机 DIR=PD4 在用)
    - 结论: GPIOD 全端口不可用于 GPIO 输入 (板级/硅片级问题)
    - 历史佐证: laser.h "TOF200F 不可使用 PD6 作为 RX 引脚"

  【软件修复 (全部验证通过)】
    P1 - RX 3x 过采样: state 1 tick==1→2, 采样点从 3tick→4tick
    P2 - RX 上拉: GPIO_PUPD_NONE → GPIO_PUPD_PULLUP
    P3 - TX 超时诊断: sw1/2_tx_timeout 计数器
    P4 - printf 格式: %lu → %u (避坑 MicroLIB)
