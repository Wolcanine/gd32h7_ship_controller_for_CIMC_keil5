一、项目概览
================================================================================

项目名称    水面垃圾收集船主控程序
目标平台    GD32H759IMK6 (ARM Cortex-M7, GigaDevice)
编译环境    Keil MDK5 (uVision5), ARM Compiler 5
代码仓库    本地 Git 管理，分支 master
项目路径    D:\BaiDuNetdiskDownload\04 例程模板\00-Project_Template

项目功能    遥控/自动双模水面垃圾收集船，集成以下子系统：
            - PS2 无线遥控（油门/差速/机械臂）
            - MPU6050 陀螺仪姿态感知
            - TOF200F 激光测距
            - 四电机差分推进（双引脚 H 桥 PWM 驱动）
            - 6 舵机机械臂（PCA9685 I2C 驱动 + 逆运动学）
            - GPS/北斗定位（ATGM336H-5N，SW UART CH1, 9600bps）
            - 视觉识别模块（接口预留，同板集成）
            - OLED 状态显示（SSD1306）
            - PID 航向闭环 / 自动导航

四、引脚分配（当前生效）
================================================================================

【电机驱动】双引脚 H 桥，全部在 GPIOC（软件 PWM 要求同端口）：
  D0 = PC2  (J4-32)   左侧双电机正转 PWM    → 驱动板A CH1+CH2 并联
  D1 = PC3  (J4-33)   左侧双电机反转 PWM    → 驱动板A CH1+CH2 并联
  D2 = PC5  (J4-34)   右侧双电机正转 PWM    → 驱动板B CH1+CH2 并联
  D3 = PC10 (J4-51)   右侧双电机反转 PWM    → 原 UART_CAM TX，已复用

【舵机 PCA9685】I2C（软件 I2C，与 MPU6050 共享总线）：
  SCL = PE13          软件 I2C 时钟
  SDA = PE15          软件 I2C 数据
  OE  = PC11 (J4-52)  使能端（低电平有效），原 UART_CAM RX，已复用

【传感器】：
  TOF200F 激光:  UART3,  PA0-TX(J5-44) / PA1-RX(J5-43), 115200
  调试串口:      UART4,  PB5-TX(J5-38) / PB13-RX(J5-36), 9600 → CH340 → USB
  MPU6050:       I2C,    PE13-SCL / PE15-SDA (与 PCA9685 共享)
  OLED:          另一个项目控制，本项目不管理

【PS2 遥控器】：
  DAT = PA5 (J5-42)  CMD = PA7 (J5-34)  CS = PB12 (J4-59)  CLK = PB10 (J4-31)
  注: 原 F407 方案使用 PB14/PB15/PE5/PE6，但此四脚核心板未引出，
      已改用上述引脚（参考 ps2.h 2026-06-10 修改记录）

【步进电机传送带】3 线 (EN/DIR/STEP)：
  STEP = PG3 (J4-57)   步进脉冲, TIMER6 @ 2kHz
  DIR  = PD4 (J4-22)   方向 (H=正转)
  EN   = PH6 (J4-37)   使能 (H=使能)
  注: 原 PA2/PD6 已腾给预留串口, 后因 PD6(GPIOD) 输入不可用而弃用

【SW UART 软件串口】两路全双工 (2026-06-18 新增，2026-06-18 改 9600bps)：
  CH1: PE2-TX(J5-56) / PE5-RX(J5-58)  9600, GPIOE, TIMER1 @ 28.8kHz 【GPS 占用】
       GPS模块接线: VCC→5V, GND→GND, TX→PE5, RX→PE2, PPS→NC
  CH2: PF7-TX(J4-17) / PF6-RX(J4-19)  9600, GPIOF, TIMER1 @ 28.8kHz 【预留】

【预留硬件串口】(因 GPIOD/GPIOF RX 硬件不可用, 暂由 SW UART 替代)：
  USART1: PA2-TX(J4-21) / PD6-RX(J4-23)  — RX 不可用
  UART6:  PF7-TX(J4-17) / PF6-RX(J4-19)  — 被 SW UART2 占用

【已释放可复用引脚】：
  PA2 (J4-21), PD6 (J4-23) — 原 USART1/SW UART1, PD6 输入不可用, 已释放
  PF9/PD2/PC12 — 已被另一工程触摸屏 XPT2046 占用

【⚠ GPIOD 端口限制 (2026-06-18 确认)】：
  GPIOD 全端口不可用于 GPIO 输入 (仅输出和同板自回环可用)
  已确认: PD4/PD6 外接信号均失败(PD4崩溃/PD6无数据)
  输出正常: PD4=步进DIR 正常工作中

五、关键架构决策与约束
================================================================================

1. 电机驱动板
   - 原 IN1+IN2+ENA 三线制驱动板已更换为双引脚 H 桥驱动板
   - 控制逻辑：正转=(PWM,0) 反转=(0,PWM) 滑行=(0,0) 刹车=(1,1)
   - 同侧两个电机并联：一个驱动板同时驱动左侧/右侧两个电机
   - 安全限幅：PWM_MAX_DUTY=25%，测试用 20% 油门上限

2. 软件 PWM
   - 使用 gd32_driver_pwm 软件 PWM 模块（非硬件定时器 PWM）
   - 约束：所有电机引脚必须在同一 GPIO 端口 → 4 引脚全部选在 GPIOC
   - 频率：200Hz（周期 5ms）

3. 主循环
   - 50Hz 控制周期（TIMER3 20ms 定时器）
   - 带看门狗超时保护：超过 500ms 无周期信号 → 紧急停车
   - 串口传感器上报频率：1Hz（每 50 帧打印一次），便于串口助手观察

4. 视觉模块
   - 原方案为双 MCU（H7+F4），UART7 板间通信
   - 比赛规则要求仅使用 H7，改为同板集成
   - vision.h/c 为占位实现，数据结构已定义，等待实际摄像头接口对接
   - UART_CAM (UART7, PC10/PC11) 已废弃，引脚已释放

5. PCA9685 OE 使能 + MODE2 推挽输出 + 舵机校准
   - OE 引脚（PC12）低电平有效
   - 机械臂空闲时自动关闭 PWM 输出（省电 + 避免舵机抖动）
   - MODE2: OUTDRV=1(推挽)+OCH=1(STOP更新), 回读验证
   - 校准: 角度即舵机角度, 修改默认位置(action_angle)即可, 无需 offset
     ch0=270°舵机, ch1~5=180°; 夹爪限位 51°~141°
   - PS2 START 循环预设XY坐标 → IK解算 → 缓冲插值移动
   - PS2 SELECT 回 PARK (与上电默认一致)
   - 串口 1Hz 上报 6路角度+脉宽μs

6. 电机 PWM 双引脚 H 桥 + 油门分层控制
   - 四路 PWM 全部在 GPIOC: D0=PC2, D1=PC3, D2=PC5, D3=PC10
   - 软件 PWM 200Hz (TIMER4), 正向HIGH=有效
   - 油门分层 + 启动突加 + 方向切换刹车保护

7. 软件 UART (sw_uart)
   - TIMER1 @ 28.8kHz (3x 过采样 = 9600 × 3)
   - 两路全双工，同一 ISR 处理收发状态机
   - CH1 分配 GPS 模块 (ATGM336H-5N), CH2 预留
   - RX: 下降沿检测 + 3 倍过采样 (start bit 中心验证)
   - TX: 精确位定时，LSB first
   - 使用 GPIO_ISTAT 直读 + gpio_bit_set/reset 直写
   - printf 格式必须用 %u 而非 %lu (MicroLIB 兼容)

7. GPIOD 端口限制
   - GPIOD 全端口不可用于 GPIO 输入 (仅输出可用)
   - 所有输入引脚避开 GPIOD

8. 编译注意事项
   - GD32H7 标准库没有 GPIO_OSPEED_2MHZ 宏（仅支持高速），使用 GPIO_OSPEED_60MHZ
   - ARM Compiler 5 不完全支持 printf 中的 UTF-8 多字节字符，请使用 ASCII 替代
   - Keil 需勾选 "Use MicroLIB" 以使用 printf 重定向
   - MicroLIB 的 printf 对 %lu 支持有限，使用 %u + (unsigned) 强制转换

六、测试模式
================================================================================

main.c 顶部有六个测试宏，同时只启用一个。全部注释 = 正常全系统模式。

  //#define MOTOR_MODULE_TEST       四电机 PWM + PS2 遥控
  //#define PCA9685_MODULE_TEST     舵机 PCA9685 + PS2 遥控
  //#define DIRECT_SERVO_SWEEP     舵机扫频 (PF9, 50Hz PWM)
  //#define SW_UART_ECHO_TEST       SW UART 两路 PC 回显测试 (9600bps)
  //#define GPS_MODULE_TEST         GPS 模块接收测试 (SW UART CH1, 9600bps)
  //#define STEPPER_MODULE_TEST     步进电机传送带测试 (PS2 L1/L2)

MOTOR_MODULE_TEST:
  - PS2 ↑↓ 油门 +/-1%, □○ 油门 +/-5%, ←→ 差速, × 回正, △ 急停
  - 油门映射: 用户 0→停止, >0→内部 10%~100% (motor_throttle_map)
  - 启动突加: 静止→转动先给 70%×100ms，克服静摩擦

PCA9685_MODULE_TEST:
  - 7通道固定角度输出 (ch8~ch14: 0°~180°), 避开机械臂 ch0~ch5

DIRECT_SERVO_SWEEP:
  - PF9 直驱 50Hz PWM 慢速扫频, PCA9685 ch8~ch14 固定角度参考

SW_UART_ECHO_TEST:
  - SW UART CH1(PE2/PE5) + CH2(PF7/PF6) PC 回显
  - 每秒调试串口打印统计: rx/tx/edge/timeout/ISR

GPS_MODULE_TEST:
  - SW UART CH1 (PE2/PE5) 连接 ATGM336H-5N GPS 模块 @ 9600bps
  - 每秒打印 NMEA 解析状态: 定位状态/UTC时间/经纬度/速度/航向/卫星数
  - 模块板载 LED 闪烁 = 已定位，常亮/灭 = 未定位
  - 冷启动首次定位需 30s~数分钟，需在开阔地/窗边测试

STEPPER_MODULE_TEST:
  - PS2 L1=正转(收垃圾), L2=反转(释放), 松手=停止
  - 每秒打印累计步数
  - 引脚: STEP=PG3 DIR=PD4 EN=PH6, TIMER6 @ 2kHz
  - Stepper_SetSpeed 已做幂等保护：dir/speed_div 未变则跳过重置，避免 50Hz 循环打断脉冲序列
